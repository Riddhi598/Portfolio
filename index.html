Project: DHCP & Wireshark Analysis

Specialized Cyber Security Deep-Dive

Project Overview

Designed and configured a dynamic network architecture in Cisco Packet Tracer to automate client IP address allocation. Captured and analyzed the underlying DORA process via Wireshark to verify packet-level compliance and network efficiency.


Problem Statement

An enterprise network contains a legitimate DHCP server and a rogue DHCP server. If clients accept offers from the fake server, attackers can manipulate IP settings, reroute traffic, or block network access — causing outages and interception risks.

Objectives

  • Understand the working of DHCP in enterprise networks.
  • Understand the working of DHCP in enterprise networks.
  • Simulate a Rogue DHCP Server attack.
  • Analyze DHCP traffic using Wireshark.
  • Implement DHCP Snooping to improve network security.

Background

Dynamic Host Configuration Protocol (DHCP) automatically assigns IP addresses, subnet masks, gateways, and DNS settings to devices. It centralizes address management for faster provisioning and consistent network configuration across large enterprise environments.

Tools Used: Cisco Packet Tracer, Wireshark, NetworkPM.jpeging protocols

Key Takeaways: Detailed understanding of the DORA (Discover, Offer, Request, Acknowledge) process, analyzing packet structures, and ensuring network broadcast safety.

SDG mapping

This project aligns with Sustainable Development Goal 9: Industry, Innovation and Infrastructure. By securing DHCP services and strengthening enterprise network resilience, we support scalable, reliable infrastructure and protect digital services that underpin economic activity.

Conclusion

This project demonstrated the implementation of a secure enterprise DHCP network using Cisco Packet Tracer. It highlighted the risks posed by Rogue DHCP Server attacks and showed how DHCP Snooping can effectively protect enterprise networks by allowing only trusted DHCP servers to assign IP addresses.
